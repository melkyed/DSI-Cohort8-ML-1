## 1. Project Overview

**Business Problem**
Marketing campaigns conducted through phone calls require significant time, effort, and financial resources. However, only a small percentage of clients actually subscribe to the term deposit product. As a result, the bank spends resources contacting many customers who are unlikely to be interested.

The goal of this project is to build a machine learning model that predicts whether a bank client will subscribe to a term deposit based on demographic information, financial characteristics, and previous campaign interaction history. By identifying clients who are more likely to subscribe before contacting them, the bank can:

- Increase campaign efficiency and reduce unnecessary calls
- Improve conversion rates and optimise the allocation of marketing resources
- Understand which factors most influence whether a client says yes
- Make more informed decisions about future campaigns

The dataset mixes demographic, financial, and campaign-related attributes. Many entries contain the placeholder `"unknown"`, especially in fields like `education`, `contact`, and `poutcome`.


> **Why this step is needed:** Without registering the kernel, Jupyter uses the system Python and cannot see the packages installed in `ml-team1-env`. Always select `Python (ml-team1-env)` as the kernel when opening the notebooks.

---

### Step 5 — Dataset

`data/raw/bank-full.csv` is already included in this repository — no separate download is needed.

If you ever need to retrieve the original file, it is available from the UCI ML Repository:
[https://archive.ics.uci.edu/dataset/222/bank+marketing](https://archive.ics.uci.edu/dataset/222/bank+marketing)

> **Important:** The UCI repository offers two files — `bank.csv` (10% sample) and `bank-full.csv` (full 45,211-record dataset). This project uses `bank-full.csv`. 4. Runs full pipeline: encoding → splitting → training → evaluation → SHAP → business impact

## 5. Methodology

**Primary Objective:** Develop and evaluate a predictive ML model that determines the likelihood of a client subscribing to a term deposit using demographic, financial, and historical campaign data.

**Secondary Objectives:**
- Identify and rank the most influential customer attributes driving subscription decisions
- Isolate high-probability customer segments with conversion rates substantially above the historical baseline
- Analyse the precision-recall trade-off and establish an optimal classification threshold aligned with business cost-benefit goals

### 5.1 Problem Framing

Binary classification: output is either `1` (subscribed) or `0` (did not subscribe). Subscription probability is also used directly to enable threshold tuning based on business cost-benefit analysis.

### 5.2 Data Preparation

| Step | Action |
|---|---|
| Leakage removal | Drop `duration` — only measurable after the call ends |
| EDA segments dropped | Drop `age_group`, `balance_group`  derived for EDA only, redundant with raw columns |
| Binary encoding | `default`, `housing`, `loan` → 0/1 (applied before `get_dummies`) |
| One-hot encoding | `job` (12), `marital` (3), `education` (4), `contact` (3), `month` (12), `poutcome` (4) with `drop_first=False` |
| Feature count | 16 raw columns → 47 model features |
| Split | Stratified 70 / 15 / 15 with `random_state=42` at both steps |
| Scaling | `StandardScaler` on Logistic Regression inputs only  fit on train, transform val/test |
| Imbalance | `class_weight="balanced"` for LR and RF; `scale_pos_weight=7.55` for XGBoost |
| See [`notebooks/README.md`](notebooks/README.md) for the full pipeline details.|

### 5.3 Models & Evaluation

Three models evaluated in order of complexity:

| Model | Role | Key parameters |
|---|---|---|
| Logistic Regression | Interpretable baseline | `class_weight=balanced`, `C=1.0`, `max_iter=1000`, `random_state=42` |
| Random Forest | Ensemble benchmark | `n_estimators=200`, `max_depth=15`, `class_weight=balanced`, `random_state=42` |
| XGBoost | Selected model | `n_estimators=300`, `lr=0.05`, `max_depth=4`, `scale_pos_weight=7.55`, `random_state=42` |
| See [`models/README.md`](models/README.md) for more on model selection rationale, evaluation metrics, and explainability approach.|

**Evaluation metrics:** ROC-AUC, F1-Score, PR-AUC, Precision, Recall, Confusion Matrix.
**Threshold tuning:** Grid search over 0.05–0.95 on validation set, maximising F1.

### 5.4 ML System Design

| Consideration | Approach |
|---|---|
| **Reliability** | Return calibrated probabilities; flag low-confidence predictions |
| **Fairness** | Audit predictions across `age`, `marital`, `job` to detect discriminatory patterns |
| **Adaptability** | Document retraining strategy based on performance monitoring as distributions shift |
| **Business vs ML Objectives** | ML objective = maximise F1/AUC. Business objective = increase campaign ROI. Quantify cost savings from targeting top 20% vs random outreach |

### 5.5 Deliverables

- Exploratory Data Analysis report with visualisations
- Reproducible preprocessing pipeline (relative paths, documented run order)
- Trained and tuned models with performance comparison
- SHAP-based feature importance analysis
- Segment and decile analysis lift by probability decile
- Business impact summary: projected ROI improvement from targeted outreach
- Fairness audit across demographic groups
- Final project report and presentation deck

## 6. Results Summary

| Model | ROC-AUC | PR-AUC | F1 | Precision | Recall | Threshold |
|---|---|---|---|---|---|---|
| Logistic Regression | 0.760 | 0.381 | 0.422 | 0.398 | 0.450 | 0.61 |
| Random Forest | 0.795 | 0.417 | 0.456 | 0.450 | 0.462 | 0.60 |
| **XGBoost ✓** | **0.802** | **0.437** | **0.459** | **0.469** | 0.450 | 0.17 |

XGBoost selected — wins on 4 of 5 metrics and produces fewest false positives (404 vs 447 RF, 541 LR).

**Business impact — Top 20% targeting strategy:**

| Strategy | Calls | Profit | Margin |
|---|---|---|---|
| Random outreach (all clients) | 45,211 | EUR 115,185 | 17% |
| Machine Learning — Top 30% | 13,563 | EUR 384,855 | 189% |
| Machine Learning — Top 20% ✓ | 9,042 | EUR 383,070 | 282% |

+233% profit improvement · 80% fewer calls · 65.4% of subscribers captured

## 7. Risks & Unknowns

| Risk | Detail | Mitigation |
|---|---|---|
| **Class Imbalance** | Only 11.7% of records are positive. Accuracy is misleading. | Use PR-AUC and F1. Apply class weights and threshold tuning. |
| **Data Leakage** | `duration` is only known after the call ends. | Excluded from all models before any preprocessing step. |
| **Representativeness** | Single Portuguese bank, May 2008 – Nov 2010. | Document limitations; treat findings as bank-specific. |
| **Proxy & Fairness** | `age`, `job`, `marital`, `education` can proxy protected characteristics. | Fairness audit across demographic groups included in modelling notebook. |
